# API Reference

## REST Endpoints

### Variant Analysis

- `GET /variant/{genome_build}/{variant_id}` - Get PVS1 analysis for variant
- `GET /variant/search?q={gene}&genome_version={build}` - Search variants

### CNV Analysis

- `GET /cnv/{genome_build}/{cnv_id}` - Get PVS1 analysis for CNV

### System Management

- `GET /health` - Health check with component status
- `GET /metrics` - Prometheus metrics (gated by
  `AUTOPVS1_LINK_METRICS_ENABLED`, default `true` for the HTTP server,
  `false` for stdio MCP)
- `GET /api/cache/stats` - Cache performance metrics (HTTP-only)
- `POST /api/cache/clear` - Clear all caches (HTTP-only)

OpenAPI documentation is available at `/docs` and `/redoc` when the FastAPI
server is running.

## MCP Integration

### Available Tools

- `get_variant_pvs1_data(genome_build, variant_id)` - Research-use PVS1 analysis for an AutoPVS1 SNV/indel ID such as `X-82763936-A-T`.
- `get_cnv_pvs1_data(genome_build, cnv_id)` - Research-use PVS1 analysis for an AutoPVS1 CNV ID such as `17-15000000-20000000-DEL`.
- `search_variants(query, genome_build=None, limit=10, cursor=None, genome_version=None)` - Search AutoPVS1. Use `genome_build`; `genome_version` is a deprecated alias for one release.
- `get_server_capabilities()` - Compact MCP discovery payload with a pointer to `autopvs1-link://capabilities`.
- `clear_cache()` - Cache management. **Gated** by `AUTOPVS1_LINK_ENABLE_DESTRUCTIVE_TOOLS=true`; default disabled.

All MCP tools return structured content with the standard envelope:
`ok`, `data`, `error`, and `meta`. `meta.research_use_only` is always `true`,
and `meta.recommended_citation` cites AutoPVS1. Expected validation and
upstream failures use `ok: false` with stable `error.code` values rather than
raw HTML, tracebacks, or JSON-RPC protocol failures.

Search pagination uses `limit` and `cursor`. The first call may omit `cursor`;
when `data.next_cursor` is present, pass that value as the next call's
`cursor`. `limit` defaults to 10 and is bounded to the server-supported range.

The auto-generated tool catalog with full input/output schemas lives in
[`mcp-tool-catalog.md`](mcp-tool-catalog.md) and is regenerated by
`scripts/generate_mcp_tool_catalog.py`.

### Available Resources

- `autopvs1-link://cache/statistics` - Read-only snapshot of in-memory cache
  statistics.
- `autopvs1-link://capabilities` - Detailed MCP usage guidance, examples,
  error-envelope details, cache semantics, research-use framing, and citation.

The cache statistics resource always returns the configured method keys
`get_variant_data`, `get_cnv_data`, `search_variants`,
`search_with_redirect_detection`, and `resolve_hgvs_notation`, even when
counters are zero. Hits, misses, errors, evictions, total requests, hit rate,
timing, uptime, cache-key shape, and method descriptions are method-keyed so
clients can compare snapshots without rediscovering the schema.

AutoPVS1-Link MCP outputs are research-use AutoPVS1 data, not clinical
decision support. Cite AutoPVS1 as: Xiang J, Peng J, Baxter S, Peng Z.
AutoPVS1: An automatic classification tool for PVS1 interpretation of null
variants. Human Mutation. 2020;41(9):1488-1498. DOI: `10.1002/humu.24051`.

### Transport Modes

- **STDIO** - For Claude Desktop integration: `python mcp_server.py` or
  `autopvs1-link mcp`.
- **Streamable HTTP** - For web-based AI integration. Mounted under `/mcp`
  on the FastAPI app. The legacy SSE transport is retired in FastMCP 3.
- **Unified** - Both REST and MCP on the same server (default).

See [MCP_CONNECTION_GUIDE.md](MCP_CONNECTION_GUIDE.md) for client setup
instructions.
